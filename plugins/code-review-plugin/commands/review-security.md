# Review Security

## Description

OWASP Top 10に基づくセキュリティ脆弱性をチェックし、改善提案を行うコマンドです。

## Prompt Template

`security-reviewer`エージェントを使用して、セキュリティレビューを実行してください。

以下のタスクを実行してください：

1. **スキルの参照ファイルを使用してレビュー基準を取得する**

   - `security-review-skill:security-patterns` を取得し、レビュー基準を確認

2. **Critical観点でチェックする**

   - SQLインジェクション: パラメータ化クエリの使用
   - XSS脆弱性: ユーザー入力のエスケープ
   - 認証バイパス: 認証チェックの実装
   - 機密情報漏洩: ハードコードされた秘密情報

3. **High観点でチェックする**

   - 権限管理: 認可チェックの実装
   - 暗号化: パスワードハッシュ化、安全なアルゴリズム
   - セッション管理: 固定セッション、有効期限

4. **Medium観点でチェックする**

   - セキュリティヘッダー: CSP、HSTS、X-Frame-Options
   - ログ設定: 機密情報のログ除外
   - レート制限: DoS対策

5. **改善提案を提供する**

   - 脆弱なコードと修正後のコード例を提示
   - 重要度別に分類して報告
   - 各改善のリスクと対策を説明

## Notes

- 実際の攻撃コードは提供しない
- 脆弱性がない場合はその旨を報告
- 既存のセキュリティフレームワークを尊重
- レビュー結果は日本語で提供
