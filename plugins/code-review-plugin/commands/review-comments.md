# Review Comments

## Description

コード内のコメント（コメント行、JSDoc、インラインコメントなど）をレビューし、不要、古い、または問題のあるコメントを特定するコマンドです。

## Prompt Template

`comment-reviewer`エージェントを使用して、コード内のコメントをレビューしてください。

以下のタスクを実行してください：

1. **スキルの参照ファイルを使用してレビュー基準を取得する**

   - `code-review-skill:comment-review` を取得し、レビュー基準を確認

2. **コード内のコメントを分析する**

   - 対象ファイルまたはディレクトリのコードを読み込み
   - 単行コメント、ブロックコメント、ドキュメントコメントを特定
   - 各コメントを評価基準に照らして評価

3. **問題のあるコメントを分類する**

   - 不要なコメント（コードが自明な場合）
   - 古いコメント（実装と矛盾）
   - 品質の低いコメント（曖昧、一般的すぎる）
   - 形式が不適切なコメント

4. **改善提案を提供する**
   - 削除すべきコメントと更新すべきコメントを区別
   - 具体的な改善案を提示
   - 全体的なコメント規約の改善提案

## Notes

- 「なぜ」を説明するコメントは保持すべき
- 複雑なビジネスロジックやアルゴリズムの説明は価値がある
- ライセンスや著作権表示は削除しない
- 正確で有用なAPIドキュメント（JSDoc等）は保持
- レビュー結果は日本語で提供
