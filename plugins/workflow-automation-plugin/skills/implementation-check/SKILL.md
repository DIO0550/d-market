# 開発ルール準拠確認

プロジェクトの開発ルールに従って実装状況を確認するスキル。

## ワークフロー

```
1. 開発ルールの読み込み
   ↓
2. 対象コードの特定
   ↓
3. ルール準拠チェック
   ↓
4. 違反箇所の報告と修正提案
```

## 確認項目

### コード規約
- 命名規則（変数、関数、クラス、ファイル）
- ディレクトリ構造
- インポート順序
- コメント・ドキュメント

### アーキテクチャ
- レイヤー分離
- 依存関係の方向
- モジュール境界

### 品質基準
- テストカバレッジ
- エラーハンドリング
- 型安全性

## 出力形式

```
## 開発ルール準拠チェック結果

### ✅ 準拠
- {ルール1}: OK
- {ルール2}: OK

### ⚠️ 要確認
- {ルール3}: {問題箇所} - {理由}

### ❌ 違反
- {ルール4}: {違反箇所}
  - 現状: {現在のコード}
  - 推奨: {修正案}

## 推奨アクション
1. {最優先の修正}
2. {次に対応すべき項目}
```

## 開発ルールの取得

MCPツール `get_implementation_workflow` を使用して開発ルールを読み込む。

プロジェクト固有のルールがある場合は以下を確認:
- `.cursor/rules/`
- `CONTRIBUTING.md`
- `docs/development-guidelines.md`
